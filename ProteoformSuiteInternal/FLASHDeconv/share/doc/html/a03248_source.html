59  /*
60  internal details & discussion:
61 
62  We could create the list of TOPP tools and UTILS from a set of files in /share
63  instead of hard coding them here.
64  Advantage: - no recompile if new tool is added (but the new tool will necessitate that anyway)
65  - quickly changing a tool's category (e.g. from PreProcessing to Quantitation) and thus its place in TOPPAS
66  even users could rearrange tools themselves...
67  Disadvantage:
68  - when to library loads, we'd need to parse all the files. Making our start-up time even longer...
69  - when files are broken/missing, we will have a hard time initializing the lib
70  */
